Command Line

Common Commands:

  • ls -a lists all contents of a directory, including hidden files and directories
  • ls -l lists all contents in long format
  • ls -t orders files and directories by the time they were last modified
  • Multiple options can be used together, like ls -alt

From the command line, you can also copy, move, and remove files and directories:

  • cp copies files
  • mv moves and renames files
  • rm removes files
  • rm -r removes directories

Inputs:

  • standard input, abbreviated as stdin, is information inputted into the terminal through the keyboard or input device.
  • standard output, abbreviated as stdout, is the information outputted after a process is run.
  • standard error, abbreviated as stderr, is an error message outputted by a failed process.

Redirecting standard input stdin, output stdout and standard errors stderr:

  • > redirects standard output of a command to a file, overwriting previous content.
  • >> redirects standard output of a command to a file, appending new content to old content.
  • < redirects standard input to a command.
  • | redirects standard output of a command to another command.
  • cat outputs the contents of a file to the terminal.
  • echo takes a string as input and print it as output.

Examples:

  • cat oceans.txt > continents.txt         

Will overwrite the contents of continents with the ones in the ocean’s file.

  • cat glaciers.txt >> rivers.txt

Will display both the contents of glaciers with the ones in the rivers’s file.

  •  cat < lakes.txt

Will take the input from the file on the right and input it into the one on the left. Here the standard output appears in the terminal.

A number of other commands are powerful when combined with redirection commands:

  • sort : sorts lines of text alphabetically.
  • uniq : filters duplicate, adjacent lines of text.
  • grep : searches for a text pattern and outputs it.
  • sed : searches for a text pattern, modifies it, and outputs it [= Find & Replace].
  • wc : outputs the number of lines, words and characters in this order.

Examples:

  • wc plants.txt  

sort take the standard input and orders the files in lakes alphabetically and the outputs them creating a new file called sorted-lakes.txt.

  • cat lakes.txt | sort > sorted-lakes.txt

sort take the standard input and orders the files in lakes alphabetically and the outputs them creating a new file called sorted-lakes.txt.

  • sort deserts.txt | uniq > uniq-deserts.txt

 sort alphabetize the file deserts, then we pipe it to uniq and all duplicate lines are alphabetized and filtered out and then outputted in a new file uniq-deserts.txt.

  •  grep Mount mountains.txt    grep -i Mount mountains.txt

Here grep searches for “Mount” in mountains.

Meanwhile grep -i enables the command to be case sensitive, searching for capital or lowercase strings.

  •  grep -R Arctic /home/ccuser/workspace/geography

Here grep -R searches all files in a directory and outputs filename and lines containing matched results.

  •  grep -Rl Arctic /home/ccuser/workspace/geography

Here grep -Rl searches all files in a directory and outputs only filename with matched results.

  • sed 's/snow/rain/' forests.txt

Here sed searches the file forests.txt for the word “snow” while s will substitute It when found with rain.

  • sed 's/snow/rain/g' forests.txt

In this case sed searches the file forests.txt for the word “snow” and it will replace it with “rain”. All instances of “snow” will be turned into “rain”.

Each time we launch the terminal application, it creates a new session which settings and preferences are what make up the environment. And we can configure it related to the programs and the commands we create.

  •  Nano hello.txt

This command will launch the nano text editor. In the menu at the bottom  ˆ means Ctrl.

  • Ctrl + O will save the file
  • Ctrl + X will exit the nano
  • Ctrl + G opens the help menu
  • clear will clean the terminal window
  • Nano ~/.bash_profile will open the file used to store environment settings
  • ~ is the user’s home directory
  • .  indicates an hidden file
  • source ~/.bash_profile will activate the changes in the current session
  • alias command allows you to create shortcuts o aliases for commands.
  • user will configure the variable USER to the name inputted. This command allows you to create shortcuts o aliases for commands.
  • export USER=”Micaela”
  • PS1 will configure the variable that defines the markup and style of the command prompt.
  • export PS1=”>> ”
  • $PATH will show the environment variable that stores a list of directories separated by a colon:
  • echo $PATH
  • $HOME will show the home directory.
  • echo $HOME
  • env is the command that stands for environment and returns a list of the environment variables for the current user. .
  • env | grep PATH
